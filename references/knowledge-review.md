# 详细知识回顾规则

## 总结目标

最终产物不是聊天摘要，而是用户在遗忘一段时间后仍能快速恢复完整心智模型的详细知识回顾。完整读取所有有效对话和相关代码证据，去除重复后按照知识关系重写。不得因为原始对话很长而只保留结论，也不得按时间顺序拼接问答。

## 恢复完整任务谱系

优先使用 Codex 任务工具识别当前任务 ID，逐页读取当前任务，并确认是否存在 fork 父任务。任务工具不能提供完整谱系、压缩前文本或必要工具结果时，运行：

```bash
python3 scripts/extract_thread_lineage.py \
  --thread-id <当前任务ID> \
  --sessions-root /Users/ann/.codex/sessions \
  --format markdown \
  --include-tool-evidence
```

递归跟随 `forked_from_id`。读取父任务时以子任务创建时间为上界，避免混入 fork 之后父任务新增的无关内容。原始 session 只读，不得改写、移动或删除。

完整读取用户与助手的有效文本，包括响应标注、用户对先前回答的批注、代码围栏和 Mermaid 图。环境上下文、重复系统指令、纯工具协议元数据不属于学习正文。工具输出需要全部检查，但只有形成项目事实的代码片段、查询结果、配置、日志和命令结果进入知识回顾。敏感内容和与学习无关的大段输出不得复制。

如果父任务文件、压缩前记录或关键证据无法定位，把完整性状态标记为不完整，列出缺失任务或范围。不得在这种情况下宣称“已覆盖全部会话”。

## 建立覆盖清单

重组前提取并编号所有有效用户学习消息，同时读取学习面板中的 `P-NNN`、`Q-NNN`、`E-NNN`、`M-NNN` 和 `V-NNN`。为每项确定所属主线和计划正文位置。

使用下列覆盖关系验收：

```text
用户学习消息 → 知识主线 → 正文章节
重点 P-NNN → 详细解释位置
疑点 Q-NNN → 已解决正文或待验证项
证据 E-NNN → 支持的项目事实
误区 M-NNN → 对应机制章节
图表 V-NNN → 正文嵌入位置
任务谱系 → 已读取消息数量与截止范围
```

重复问题可以映射到同一章节，但每条消息都必须有归属。不能只凭最终文档篇幅判断完整性。

## 动态生成知识结构

根据机制、数据流、状态变化、业务目标和失败边界重新聚类知识。相近问题共享一个因果模型时融合；需要不同基础模型或能够独立形成完整业务链路时拆分。先生成最终知识地图，再从知识地图生成章节，不要沿用对话早期已经过时的地图。

固定章节模板只能作为兜底。核心章节数量由真实主线决定。“面试知识”“代码证据”“已纠正误区”等辅助部分不能吸收本应独立的基础知识。项目事实和建议方案必须在章节层级或明确的小节中分开。

## 详细重写每条主线

每条核心主线使用连贯、详实的中文段落，按照适合该主题的自然顺序覆盖：问题背景、核心概念、专业术语、内部机制、完整因果关系、业务运行链路、原理与代码映射、项目已有设计、已有优化逻辑、收益与代价、失败边界、用户重点、历史疑点、已纠正误区、面试追问、迁移规律和待验证事实。

不要机械地把上述内容写成十几个零散小点。应把有因果关系的内容合并为自然段，并用小标题帮助回顾。代码片段只保留理解机制或证明项目事实所需的最小完整范围，同时说明它证明什么。运行态数字必须标注为当时快照，不能写成长期固定配置。

关键 Mermaid 图必须放在相关机制首次被完整解释的位置。图前后都要有详细文字。相同图不要在多个章节重复；需要再次引用时指向原章节。

## 保留学习过程中的认知变化

最终正文吸收已经纠正后的结论，不在主体中并列保留错误答案。具有教学价值的误区应在相关机制章节解释“为什么原理解容易产生、正确模型是什么、这个纠正怎样影响业务判断”，并在“已纠正误区”索引中提供位置。

疑点已解决时写入对应正文；部分解决或缺乏项目证据时保留在待验证项中，并说明还需要什么代码、运行实验或业务信息。用户明确标记的重点应比普通内容获得更充分的解释，但不要通过无意义重复增加篇幅。

## 推荐整体结构

根据实际主线动态命名核心章节，同时保留必要的横向信息：

```text
学习背景与目标
最终知识地图
按最终地图生成的若干核心主线章节
项目已实现内容与建议方案边界
重点、疑点和已纠正误区索引
尚未确认的事实与后续验证
理解检测题与参考推理
任务谱系、来源和完整性覆盖
```

“来源和完整性覆盖”用于审计，不应承载核心知识。图表必须留在正文，而不是移入来源附录。

## 写入后核对

确认最终文件与学习面板位于同一文件夹，目标文件真实存在，知识地图与章节一致，所有核心主线都有详细解释，Mermaid 围栏成对，用户重点和疑点均有归属，项目事实具有证据，来源任务及父任务已经登记。发现缺口时先补全再报告完成。
